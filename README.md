# Medical Note Classifier

> An NLP application that classifies clinical/medical notes into medical specialties  
> (e.g. Cardiology, Neurology, Orthopedics) using a fine-tuned **BERT** transformer model.

---

## 📁 Project Structure

```
Medical Note Classifier/
│
├── data/
│   ├── mtsamples.csv          ← (place raw Kaggle dataset here)
│   ├── cleaned_data.csv       ← generated by preprocess.py
│   └── label_mapping.csv      ← generated by preprocess.py
│
├── src/
│   ├── preprocess.py          ← data cleaning & preprocessing
│   ├── train_model.py         ← fine-tuning BERT & evaluation
│   └── predict.py             ← terminal-based prediction

│
├── results/
│   └── evaluation_report.txt  ← accuracy, precision, recall, F1
│
├── requirements.txt
└── README.md
```

---

## 🎯 Objective

Given a raw clinical/transcription note, the model predicts the **medical specialty** it belongs to.

**Example**

| Input | Output |
|---|---|
| *"Patient reports severe chest pain and shortness of breath."* | **Cardiology** |
| *"MRI shows herniated disc at L4-L5 with nerve compression."* | **Orthopedics** |
| *"EEG findings consistent with generalised tonic-clonic seizures."* | **Neurology** |

## 🚀 Usage

### Step 1 – Preprocess the data

```bash
python src/preprocess.py
```

This will:
- Load `data/mtsamples.csv`
- Remove null/empty rows
- Clean and normalise text
- Filter rare specialties (< 10 samples)
- Save `data/cleaned_data.csv` and `data/label_mapping.csv`

---

### Step 2 – Train the model

```bash
python src/train_model.py
```

This will:
- Load the cleaned dataset
- Split into 80% train / 20% test (stratified)
- Fine-tune `bert-base-uncased` for 3 epochs
- Save the best model to `models/`
- Save evaluation metrics to `results/evaluation_report.txt`

> ⚠️ **GPU recommended.** Training on CPU will work but will be slow.  
> Estimated time: ~15-30 min on GPU | ~2-4 hours on CPU (depending on dataset size).

---

## 📊 Evaluation Metrics

Metrics are computed on the **hold-out test set (20%)** and saved to `results/evaluation_report.txt`.

| Metric | Description |
|---|---|
| **Accuracy** | Overall fraction of correct predictions |
| **Precision** | Weighted average precision across all classes |
| **Recall** | Weighted average recall across all classes |
| **F1 Score** | Harmonic mean of Precision and Recall |

---

## ⚙️ Configuration

Key hyperparameters in `src/train_model.py`:

| Parameter | Default | Description |
|---|---|---|
| `MODEL_NAME` | `bert-base-uncased` | Pre-trained transformer model |
| `MAX_LEN` | `256` | Max token length (BERT limit: 512) |
| `BATCH_SIZE` | `8` | Training batch size |
| `EPOCHS` | `3` | Number of fine-tuning epochs |
| `LR` | `2e-5` | Learning rate |
| `TEST_SIZE` | `0.20` | Fraction of data for testing |

## 🛠️ Troubleshooting

| Issue | Solution |
|---|---|
| `FileNotFoundError: mtsamples.csv` | Download the dataset from Kaggle and place in `data/` |
| `No trained model found` | Run `python src/preprocess.py` then `python src/train_model.py` |
| `CUDA out of memory` | Reduce `BATCH_SIZE` or `MAX_LEN` in `train_model.py` |
| Slow training on CPU | Reduce `EPOCHS` to 1 or use a smaller `MAX_LEN` (e.g. 128) |
